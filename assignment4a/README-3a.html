Image Crawler Fork-Join Pool App (Assignment 3a)

The purpose of this assignment is to deepen your experience using Java's functional programming features and their integration with Java's object-oriented fork-join framework. In particular, in this assignment you'll develop an app that uses Java's fork-join framework to obtain, store, and transform images.

Like the earlier assignments, the code you'll write here will have a GUI and will run as an Android app. Please use this GUI along with the unit tests discussed below to help test and debug your solution.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 3a, however, you only need to be aware of the following three directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers, which contains the skeleton(s) you'll need to fill in, as described below.
  • image-crawler/src/test, which contains a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignmentTests, which contains an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 29, so you will need to select an emulator that uses an API >= 29. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

In this part of the assignment you'll complete the implementation of an image crawler class in the ForkJoinCrawler1.java file that obtains, stores, and transforms images. This class uses Java's fork-join framework to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from an HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features is provided in the SequentialLoopsCrawler.java file. You will focus on converting this Java 7-based class to the ForkJoinCrawler1 class that uses Java's fork-join framework. Feel free to use Java functional programming features (such as lambda expressions and method references) and sequential streams (but not parallel streams) in your ForkJoinCrawler1 class implementation.

Assignment 3a is designed to give you experience programming directly to Java's fork-join framework APIs.
