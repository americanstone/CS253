Image Crawler Stream App (Assignment 2a)

The purpose of the first part of this two-part assignment is to deepen your experience using Java functional programming features and their integration with Java sequential streams. In particular, in this assignment you'll develop an app that uses Java sequential streams to obtain, transform, and store images.

The code you'll write for this assignment has a GUI and runs as an Android app. Please feel free to use this app to help debug your solution.

The following resources may be helpful in completing this assignment:

  • Detailed coverage of Java sequential streams is available in weeks 4 and 5 in our course website.

  • Good overviews of Java streams appear here and here. I highly recommend you get the book Modern Java in Action to learn more about Java streams programming.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 2a, however, you only need to be aware of the following directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils and image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers, which contains the skeleton(s) you'll need to fill in, as described below.
  • image-crawler/src/test, which is a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignmentTests, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 29, so you will need to select an emulator that uses an API >= 29. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

In this part of the assignment you'll complete the implementation of an image crawler class in the image_crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/SequentialStreamsCrawler.java file that obtains and stores images. This class uses Java sequential streams to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from HTML pages reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features is provided in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/SequentialLoopsCrawler.java file. You will convert this Java 7-based class to the SequentialStreamsCrawler class that uses Java sequential streams.

Undergraduate students must do the following:

  • Implement the key methods in the Spliterator interface in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArraySpliterator.java class (which was done by graduate students in assignment 1b). You'll need these methods for assignment 2b, so now is a good time to implement them.

Likewise, graduate students (or students who are taking this course for graduate credit) must do the following additional work:

  • Implement the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/SynchronizedArray class, which will then be used in the assignment 2b's ArrayCollector class (not included in this assignment). Undergraduate students can ignore the SynchronizedArray class.

Your solution should use no loops or if/else statements in this assignment and instead use Java sequential stream aggregate operations.
