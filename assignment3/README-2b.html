Image Crawler Parallel Stream App (Assignment 2b)

The purpose of this assignment is to deepen your experience using Java's functional programming features and their integration with Java's parallel streams. In particular, in this assignment you'll develop an app that uses several Java parallel streams strategies to obtain, transform, and store images.

The code you'll write for this assignment has a GUI and runs as an Android app. Please feel free to use this app to help debug your solution.

  • Good overviews of Java sequential and parallel streams appear here and here. I highly recommend you get the book Modern Java in Action to learn more about Java streams programming.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 2b, however, you only need to be aware of the following directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils and image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers, which contains the skeletons you'll need to fill in, as described below.
  • image-crawler/src/test, which is a set of unit tests that exercise the features you're implementing and are used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignmentTests, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 29, so you will need to select an emulator that uses an API >= 29. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

In this part of the assignment you'll start by completing the implementation of an image crawler class in the ParallelStreamsCrawler1.java file in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers folder. This class applies a couple of minuscule tweaks to SequentialStreamsCrawler class so that Java parallel streams are used to perform an "image crawl" starting from a root Uri to obtain, transform, and store images. Depending on the parameters used to run the tests, images from HTML pages reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

Also in this part of the assignment you'll complete the implementation of an image crawler class in the ParallelStreamsCrawler2.java file in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers folder. This classes use a more advanced Java parallel stream implementation to perform an "image crawl" starting from a root Uri to obtain, transform, and store images.

The bulk of the code for the app is provided in the skeletons. Your should convert your SequentialStreamsCrawler class from assignment 2a to the ParallelStreamsCrawler1 class in assignment 2b by making minuscule changes to use Java parallel streams. You'll also need to complete the "TODO" tasks in the ParallelStreamsCrawler2.java file, which is a variant of the ParallelStreamsCrawler1.java file that uses Java parallel streams in a more sophisticated manner.

You must also complete the collector implementation in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArrayCollector.java file. Graduate students should implement a concurrent collector using the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/SynchronizedArray class implementation from the last assignment in their ArrayCollector.java implementation. Undergraduate students can ignore the SynchronizedArray class and implement a non-concurrent collector by using their UnsynchronizedArray class.

Your solution should use no loops or (unnecessary) conditional statements in this assignment and instead use Java parallel stream operations and functional programming features.

Unit Tests

The unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You'll need to learn how to use the Android Studio GUI to run the unit tests locally on your computer, as described here. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the unit tests to detect problems in your code.
